php开发环境配置 & sniffer部署

首先需要启用IIS服务,添加网站,将web应用部署在服务器上,就==能看HTML==了

IIS这个Windows自带的Web服务器,方便又好用!

main.html : post请求将登录表单提交到login.php中

使用IIS部署web应用一开始是无法运行php的,需要

  1. 下载PHP里的zip文件,解压
  2. 将文件夹路径添加到环境变量中
  3. 创建php.ini-development的副本,重命名为php.ini
  4. IIS中处理程序映射——添加模块映射

配置完就==能执行php文件了==

vscode调试php:下载对应版本的Xdebug(TS/NTS)/把phpinfo输入这里让他给你建议Xdebug: Support — Tailored Installation Instructions,在php.ini里加配置↓(我设置的端口9003 & 最好写完整的dll路径),加完了看看phoinfo里有没有xdebug,有就装成了

vscode php debug插件,改setting.json(端口要一致)

配完就==能调试php了==

login.php : 连接数据库,获取用户输入,查询数据库,处理查询结果

phpinfo里没有mysqli是连不上mysql的,要开启mysqli扩展 1 2 3

php缺少mysqli扩展的原因：

  1. 配置文件php.ini中没有开启php_mysqli.dll扩展；加一句extension=php_mysqli.dll/去掉;extension=php_mysqli.dll前的;号

  2. extension_dir的地址没有指向ext目录； 修改php.ini extension_dir = “D:\install\php-8.2.5-nts-Win32-vs16-x64\ext”

  3. php_mysqli.dll文件不存在。

还要把configuration.json里的database内容改成自己数据库的名字,密码

现在==能连接mysql了==

windows上sudo要去掉,LMs.py里sklearn.linear_model.logistic要改

choose_model里model有PRIMARY KEY约束,一个模型名字只能插入一次,不然就报错,在MySQL中重复的插入数据将insert into改成insert ignore into

要windows vscode调试的话，sniffer local里的路径，还有utils里的路径的../都要改成.\\

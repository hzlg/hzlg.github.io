3.LVM 的LBA

逻辑区块地址(Logical Block Address, LBA)是描述计算机存储设备上数据所在区块的通用机制，一般用在像硬盘这样的辅助记忆设备。LBA可以意指某个数据区块的地址或是某个地址所指向的数据区块。

fdisk -l

4.检材1中网站www.kkzjc.com对应的 Web 服务对外开放的端口

在nginx取证中应该先阅读总配置文件nginx.conf以及各个子目录的配置文件

netstat -apn可以查找Linux网络状态,一般前几个就是网络服务器监听端口

7.嫌疑人登录使用的 IP 地址

last作用是显示近期用户或终端的登录情况。通过last命令查看该程序的log，管理员可以获知谁曾经或者企图连接系统。

执行last命令时，它会读取/var/log目录下名称为wtmp的文件，并把该文件记录的登录系统或终端的用户名单全部显示出来。默认显示wtmp的记录，btmp能显示的更详细，可以显示远程登录，例如ssh登录。

第一列：用户名
第二列：终端位置（pts/0伪终端，意味着从SSH或telnet等工具远程连接的用户，图形界面终端归于此类。tty0直接连接到计算机或本地连接的用户。后面的数字代表连接编号）
第三列：登录IP或内核（如果是：0.0或者什么都没有，意味着用户通过本地终端连接。除了重启活动，内核版本会显示在状态中）
第四列：开始时间
第五列：结束时间（still login in尚未退出，down直到正常关机，crash直到强制关机）
第六列：持续时间
指定显示记录的数量（显示记录中最后登录的数量）

第三部分(服务器-难点)

网探:想用工具去连接服务器的话(不管是网探还是xshell啥的)一般都是连22端口,因为这些工具的基本逻辑即使去连ssh的服务,ssh的服务默认在22端口上

有些时候会设个门槛,至少让你扫到ssh在哪

这一题里netstst -anpt看linux网络状态,看到ssh的端口是7001

题目里有网站设置,刚刚找ssh的时候看到了nginx

vim /etc/nginx/nginx.conf

看nginx的配置文件:配置了虚拟主机目录,主配置文件里面没有配置网站,加了一个include …路径

进入这个路径之后ls,看到这个路径下配了三个网站,因为配置文件里是.com的配置文件,所以default.conf是不算的,三个.com结尾的才算

仅允许域名访问,不能ip访问

下面的proxypass是关键,把请求转发给本地的8091端口,其他就没啥了,这就是一个纯跳板,纯转发用的

netstat -anpt看一下8091端口但是没有,因为docker服务没有启动

设计的时候想:首先history要看,今年的history没删,火眼,网探,命令,都能看

把history遍历一遍,就会发现docker

然后systemctl start docker

docker ps ,再netstat -anpt

8091端口就出来了

docker出来之后进入容器,第一步还是history,看到里面还有nginx,所以vim /etc/nginx/nginx.conf

主配置文件也是缺省的,include了一个路径,进入这个路径hl.conf

监听着docker内部的80端口,转发到192.168.1.176:80

看谷歌能看到第六题

找到windows的hosts:windows/system32/drivers/etc/hosts

就是管理员配了个域名,拿域名去管理网站

一定想要在检材1里找到的话,首先跟www.kkzjc.com没关系,管理员直接访问8091,就是直接访问第二级代理

docker里的nginx配置文件里有一个access log

先看下hl.conf,这里面的accesslog 是没有开的,是注释掉的,这个没有配就以主配置文件为主

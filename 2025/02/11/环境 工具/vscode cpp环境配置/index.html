windows 下的编译器选择

Pre-built Toolchains - mingw-w64

MinGw vs Cygwin vs MSVC

Cygwin 全面模拟了 Linux 的接口, MinGw 包含轻量级的 Windows 版 GCC 和其他 GNU 程序开发工具，MinGw 可以结合更现代的 MSYS2(提供了类似 Linux 的 Bash 环境，方便使用 GNU 工具链，改进了对 Windows API 的支持)，MinGW 和 Cygwin 都可以用来跨平台开发, MSVC 则用于 Windows 原生开发、游戏

gcc vs llvm

LLVM 的 编译速度快，尤其适用于需要频繁编译的开发流程（如大型 C++ 项目）。Clang 的 错误和警告信息更清晰，更容易理解，而 GCC 的错误信息往往过于复杂。Clang 内置的 静态分析工具更强大，如 clang-tidy 可以自动修复代码问题。

编译器 优点 缺点 适用场景
Cygwin 完整 POSIX 兼容层，适合 Unix 迁移 运行依赖 cygwin1.dll，性能略低 Unix 程序移植
MinGW 轻量，支持 Windows API 维护较少，GCC 版本较老 轻量级 C++ 项目
MSYS2+MinGW 现代 GCC，POSIX 支持好 需要额外配置环境 需要 GNU 工具链的项目（Qt）
LLVM-MinGW Clang 优化，兼容 MinGW，性能更好 部分 MinGW 代码兼容性问题 现代化开发，优化需求高
Clang 编译快，优化强，支持 MSVC 兼容 可能对部分 Windows 代码兼容性差 需要 Clang 高级优化
MSVC 官方支持，优化针对 Windows，兼容 WinAPI GCC/Clang 兼容性较差 Windows 原生开发、游戏（Direct X，.NET，UWP）

与调试器共舞 - LLDB 的华尔兹

只安装 clang 是不能编译的, Clang 在 Windows 上通常依赖于 Microsoft 的编译工具链，包括链接器和标准库

clang 的 target 是 msvc, 要想在 MinGW 模式下运行，可以通过添加 target 选项来编译，例如：--target = x86_64-w64-windows-gnu 或 --target = x86_64-w64-mingw 将 target 改为 mingw，不改的话，cpptools 默认的 task.json 和 launch.json 会调用 lldb 来调试，会报错 Unable to start debugging. Unable to establish a connection to LLDB. Debug output may contain more information. 安装 codelldb 可能可以解决

clangd

LSP（Language Server Protocol，语言服务器协议）是一个由微软提出的标准协议，旨在让不同的编程工具（如编辑器、IDE 等）与语言服务器进行通信，以便为开发者提供代码补全、跳转、错误检查等功能。LSP 使得编辑器和 IDE 可以通过一个统一的协议与多种编程语言的功能进行交互，而无需为每种语言分别实现不同的功能支持。

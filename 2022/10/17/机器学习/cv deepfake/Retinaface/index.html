Retinaface人脸识别

发表于 2022-10-17 | 更新于 2024-09-15

学习如何使用Retinaface进行人脸识别

Reference:

GitHub - biubug6/Pytorch_Retinaface: Retinaface get 80.99% in widerface hard val using mobilenet0.25

从零开始学人脸检测之Retinaface篇（内含魔改版GhostNet+mbv2）

Retinaface Installation

代码下载&配置:

  1. git clone https://github.com/biubug6/Pytorch_Retinaface.git

（我没git clone直接Download zip，见图）

  1. 需要Pytorch环境：Pytorch version 1.1.0+ and torchvision 0.3.0+ are needed.

  2. 需要Python3环境：Codes are based on Python 3

数据集配置:

Retinaface的github主页给了一个数据集叫 WIDER FACE

  1. WIDER FACE数据集下载链接（点进去是主页，点Home找到Download部分,分为训练40% 验证10% 和测试50%三个子集）

  2. 下载WIDER FACE 注释–百度网盘，即面部边界框和五个面部地标

  3. 下载完按照以下路径放置

  1. 新建一个py文件,执行下面这个脚本,根据数据集给出的label.txt生成wider_val.txt,具体来说就是从label.txt中提取出图片的路径放到wider_val.txt

图片识别:

图片识别需要训练好的模型和待识别的图片路径txt文件(刚刚生成的wider_val.txt)

  1. 模型配置

已经训练好的模型链接,密码fstq

下载下来之后按照这样↓放

  1. 想识别单张图片就在detect.py中修改路径然后执行即可

  1. 想识别多张图片,就在test_widerface.py中修改路径(确保路径下有路径文件,即wider_val.txt)然后执行即可

Retinaface Training

自己训练模型需要包含图片中人脸位置的label.txt,之前已经下好了

  1. 执行train.py(linux下保留CUDA_VISIBLE_DEVICES=xxx,windows下删掉)

很可惜,可能是电脑原因,或者是其他原因,我在训练时会报错RuntimeError: CUDA out of memory. Tried to allocate 50.00 MiB (GPU 0; 2.00 GiB total capacity; 1.66 GiB already allocated; 0 bytes free; 1.67 GiB reserved in total by PyTorch),也就是显存爆了

update:根据GPU memory leak while using retinaface · Issue #1025 · deepinsight/insightface对图像进行padding貌似能一定程度的解决问题 ，失败了，没看懂数据怎么处理的，代码加不进去

update：显存爆了就少搞点图片减少单次训练量即可,了解了一下batch_size,epoch,Iteration的概念

在config.py中默认batch_size为32,由于GPU的结构,设置batch为$$$$2^n$$$$可以发挥更佳的性能,我试了试设置为16,仍然爆显存了,设置为8时可以正常训练
